Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main
import (
	"fmt"
)
func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}
func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}
func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
Вывод программы:
2
1

Ключевое слово defer используется для отложенного вызова функции. 
Функция с defer всегда выполняется перед выходом из внешней функции, в которой defer объявлялась.
defer добавляет переданную после него функцию в стэк. При возврате внешней функции 
вызываются все добавленные в стэк вызовы. Поскольку стэк работает 
по принципу LIFO (last in first out), значения стэка возвращаются в порядке от последнего к первому.
Таким образом функции c defer будут вызываться в обратной последовательности 
от их объявления во внешней функции.
Аргументы функций, перед которыми указано ключевое слово defer, оцениваются немедленно. 
То есть на тот момент, когда переданы в функцию.

В данной программе сначала выполняется функция test.
Там значение x инициализируется нулевым значением при запуске функции, затем значение x меняется на 1,
а после этого в отложенной функции значение x инкрементируется. Следовательно, функция возвращает 2.
Затем выполняется функция anotherTest.
В этой функции значение x инициализируется в теле функции. Затем x передается в отложенную функцию.
Потом x приравнивается значение 1. Затем перед выходом выполняется отложенныя функция.
Так как у нее значение x изначально равнялось 0, то после завершения отложенной функции 
значение x будет 1.
```
